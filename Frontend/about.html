About PhishDefender

A real-time phishing detection system combining machine learning and explainable AI to identify and explain suspicious emails.

Developed as part of Cybersecurity and Digital Forensics Honours Project 2025/26

Introduction

Phishing remains one of the most persistent cybersecuirty threats today. Research shows that over 40% of organisations experience phishing attempts daily, with more than 75% facing attacks each month. Around 30% of phishing emails are opened, demonstrating the continued effectiveness of social engineering techniques.

Traditional detection methods struggle to keep pace. Nearly two thirds of phishing campaigns last under two hours, meaning many attacks disappear before blacklist systems can update. Initial detection rates can fall below 20%, leaving organisations exposed during the most critical early stage.

This project develops a real-time phishing detection system using machine learning combined with Explainable AI (XAI). By integrating technical and behavioural insights, the system aims to deliver accurate, transparent, and trustworthy phishing detection for modern email environments.

(De Bona & Paci, 2020; Sheng et al. 2009; Tanimu & Shiaeles, 2022)

Purpose

The aim of this project is to design and evaluate a real-time phishing detection system capable of accurately classifying malicious emails while also providing transparent, interpretable explanations for each decision.

Importance

By combining adaptive machine learning with explainability and behavioural insight, this system aims to improve trust, accountability, and practical deployment of phishing detection in modern email environments.

How To Use

PhishDefender is designed to let users connect a Gmail inbox, load emails into the system, classify them as possible phishingor legitimate messages, and then review detailed explanations for each prediction.

1. Using the website

Start on the Inbox Analysis Workspace page. If you are not already logged in, the inbox panel will display a Gmail Login form. Enter you Gmail address and you Google App Password, then press Login. Once logged in, press Refresh Inbox to load your inbox emails from the backend system (this could take some time). PhishDefender will then classify each email and display them in three main areas: the inbox list, the selected email viewer, and the two result panels for possible phishing and possible legitimate emails.

You can click any email in the inbox or result panels to open it in the viewer. The viewer shows the email subject, sender, date, prediction, confidence scores, and message content. The Analyse Results button takes you to the explanation page, where each classified email can be reviewed in more detial.

2. How the explanation page works

The explanation page is designed to show not only what the model predicted, but also why that prediction was made. Each email is displayed in its own explanation card. At the top of each card, the subject and final prediction are shown. The left side displays the original email details, including sender, date, trusted sender status, and email body. The right side contains the explanation outputs produced by the system.

The Overall explanation section gives a short human readable summary based on rule based phishing indicators, such as suspicous links, urgent wording, account related language, payment language, excessive punctuation, or unusual use of capital letters. This is intended to make the results understandable for users who may not be familiar with machine learning concepts.

The LIME summary and LIME top features sections explain which words or short phrases in that specific email had the strongest infulence on the model's prediction. LIME works locally on one email at a time, so it helps explain why that individual message was classified in a particular way.

The SHAP summary and SHAP top features section show which model features made the strongest contribution to the prediction. These features can include both TF-IDF text terms and engineered phishing indicators. Positive feature weights suggest stronger evidence toward one class, while negative feature weights suggest evidence in the opposite direction.

The confidence bars show how strongly the model learned phishing or legitimate for that email. These bars should be traeated as decision support information rather than proof that an email is definitly safe or definitly malicous.

The filter buttons at the top of the explanation page allows the user to switch between viewing all calssified emails, phishing only emails, or legitmate only email. This makes the page easier to review when many emails have been analysed.

4. How the model works

The phishing detection model is trained on multiple phishing and legitimate email datasets that are first cleaned and converted into a common format. The email text is processed using TF-IDF vectorisation, which converts words and short phrases into numerical features based on how informative they are in the dataset. This allows the model to learn common language patterns associated with phishing and legitimate emails.

In addition to TF-IDF, the system also creates several engineered features. These include link count, presence of urgent words, account related words, payment related words, number of exclamation marks, and uppercase ratio. These feautres are designed to capture behavioural warning sign often seen in phishing emails.

The final model input is created by combining the TF-IDF features with the engineered features. Several machine learning models are trained and compared, including Logistic Regression, Naive Baynes, and Support Vector Machine. The best performing model is selected using evaluation metrics such as accuracy, precision, recall, and F1-score.

During live use, each email is cleaned in teh same way as the training data, transformed into the same feature format, and passed into the saved model. The model then predicts whether the email is more likely to be phishing or legitimate. The result is shown in the interface together with the rule based sumamry, confidence scores, and explainability outputs from LIME and SHAP.
